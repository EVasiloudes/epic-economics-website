# Epic Economics Website Documentation

This directory contains all documentation, guides, and resources for the Epic Economics website project.

## 📁 Directory Structure

### `/setup`
Configuration and setup documentation for various services and integrations:
- `EMAIL_CONFIG.md` - Email service configuration
- `RECAPTCHA_SETUP.md` - Google reCAPTCHA setup instructions
- `RESEND_SETUP.md` - Resend email service setup
- `RESEND_VERIFICATION.md` - Resend verification process
- `SETUP_SECRET_KEY.md` - Secret key configuration guide

### `/design`
Design system, branding, and visual identity documentation:
- `BRAND_TOOLKIT.md` - Brand guidelines and toolkit
- `COLOR_PALETTE.md` - Color scheme and palette definitions
- `DESIGN_SYSTEM.md` - Complete design system documentation
- `epic-economics.png` - Logo and brand assets

### `/guides`
Implementation guides and best practices:
- `GSAP_OPTIMIZATION_GUIDE.md` - GSAP animation optimization
- `GSAP_PERFORMANCE_GUIDE.md` - GSAP performance best practices
- `VIDEO_IMPLEMENTATION.md` - Video integration guide
- `WEBSITE_VISIBILITY_OPTIMIZATIONS.md` - SEO and visibility optimization
- `gsap-starting-point.md` - Getting started with GSAP

### `/planning`
Project planning, strategy, and team documentation:
- `plan-with-dimis-feedback.md` - Project plan with stakeholder feedback
- `credits-bios.md` - Team member credits and biographies
- `epic-economics.md` - Project overview and description

### `/development`
Development tools, examples, and testing resources:
- `backend-example.js` - Backend implementation example
- `test-resend.js` - Resend service testing script
- `test-resend-node.js` - Node.js Resend testing
- `create_github_issues.sh` - GitHub issue creation script

### `/assets`
Documentation assets and media files:
- `/images/epic-economics-post/` - Photo assets and media files

## 🚀 Quick Start

1. **Setup**: Start with the `/setup` folder for initial configuration
2. **Design**: Review `/design` for brand guidelines and design system
3. **Development**: Check `/guides` for implementation best practices
4. **Planning**: See `/planning` for project context and team information
